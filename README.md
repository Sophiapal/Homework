Log Processing Script (Bash)

Мета роботи:
Навчитися обробляти лог-файли у Bash та працювати з командами awk, grep, sed, wc, використовуючи конвеєри.

Опис завдання:
Є файл app.log, який містить записи логів у форматі:
^<timestamp> \t ... \t <LOGGING_LEVEL> \t ...$

Де LOGGING_LEVEL може бути:
INFO
WARNING
ERROR (помилка, після якої програма може продовжувати роботу)
FATAL (критична помилка — продовження роботи неможливе)
або будь-який власний рівень логування

Завдання:
- Зчитати лог-файл app.log
- Замінити всі входження WARNING на ERROR
- Створити новий файл app_fixed.log з виправленими рівнями логування
- Порахувати частоту кожного LOGGING_LEVEL після заміни
- Вивести таблицю формату:
<LOGGING_LEVEL>: <COUNT>

Файли в репозиторії:
app.log	- Вхідний лог-файл
process_logs.sh -	Bash-скрипт обробки логів
app_fixed.log (створюється скриптом) -	Результат після заміни WARNING → ERROR

Команди, які використовуються:
grep	Виділення рядків за рівнем логування
sed	Автоматична заміна WARNING → ERROR у файлі
awk	Витягування поля з рівнем логування та агрегація даних
wc	Підрахунок кількості рядків для кожного вида grep (альтернативно до awk)

Запуск:
chmod +x process_logs.sh
./process_logs.sh

Очікуваний результат
-Створений файл app_fixed.log
-На екрані буде показано кількість логів кожного рівня, наприклад:

INFO: 12
ERROR: 9
FATAL: 1
CUSTOM_LEVEL: 3
(де всі WARNING вже пораховані як ERROR)
